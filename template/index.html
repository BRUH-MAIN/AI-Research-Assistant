01

Problem Statement

  • Slow, error-prone, and time-consuming manual processes hinder productivity in searching, analyzing, and citing research papers.
  • Such problems can be overcome by integrating AI services with the research process.
  • A high-performance database is critical to manage growing data, ensure fast queries, and support real-time AI processing for seamless user experiences.

Researchers struggle with the vast volume of academic content, manual searching, reading, summarizing papers, and properly citing sources - all of which slow down research and reduce productivity.

02

Objective

AI-Powered Platform

Develop an AI-powered research platform with capabilities for group collaboration, role-based access for mentors and researchers, and intelligent paper management.

Normalized Database

Design a normalized relational database schema in PostgreSQL to store and manage user profiles, mentor-researcher relationships, research papers, citations, and collaborative projects.

RAG Engine

Integrate the arXiv API with a Retrieval-Augmented Generation (RAG) engine to enable advanced paper exploration, conceptual querying, and automated citation preparation.

03

System Architecture

3-Service Separated Architecture

Our system implements a clean separation of concerns with three independent services:

Frontend (Port 3000)

Next.js with TypeScript
User interface & authentication

Express DB (Port 3001)

Node.js Express Server
ALL database operations

FastAPI AI (Port 8000)

Python FastAPI
AI processing ONLY

Critical Design: FastAPI has ZERO direct database access. Express server acts as the exclusive database proxy, ensuring complete separation of AI and data operations.
04

AI Implementation

Retrieval-Augmented Generation

The core AI innovation in this project is the integration of a RAG (Retrieval-Augmented Generation) architecture that:

  • Improves research accuracy by retrieving relevant academic papers
  • Processes complex queries about research concepts
  • Generates contextually accurate responses based on retrieved papers
  • Reduces hallucinations by grounding answers in source material

AI Services Architecture

The AI system is isolated from database operations for improved security and performance:

  • FastAPI AI Server handles ONLY AI processing
  • Vector embeddings for semantic search of papers
  • In-memory chat fallback when database unavailable
  • HTTP client for database persistence via Express

This separation ensures that the AI components can be scaled and optimized independently from data management operations.

RPC Integration Pattern

// Express RPC Helper Function
async function executeRPC(supabase, functionName, params = {}) {
  const { data, error } = await supabase.rpc(functionName, params);
  if (error) throw error;
  return data;
}

// Usage in Route Handler
router.get('/users/:id', async (req, res) => {
  try {
    const user = await executeRPC(supabase, 'get_user_by_id', {
      p_user_id: parseInt(req.params.id)
    });
    res.json(user);
  } catch (error) {
    res.status(500).json({ error: error.message });
  }
});
Note: All database operations use parameterized RPC calls for security and performance.

AI Service Implementation

# FastAPI Chat Service Pattern
from app.services.chat_service import chat_service
from app.services.express_client import express_client

@router.post("/{session_id}", response_model=ChatResponse)
async def send_message(session_id: str, request: ChatRequest):
    """Send message and get AI response"""
    try:
        # Process AI request in-memory
        ai_response = await ai_service.process_query(
            request.message, 
            session_id
        )
        
        # Persist to database via Express (optional)
        if express_client.is_available():
            await express_client.save_message(
                session_id, 
                request.message, 
                ai_response
            )
        
        return ChatResponse(
            response=ai_response,
            session_id=session_id
        )
    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))
